GitHub — Code Storage & Website Hosting

What It Is

GitHub is a website where teams store and share code. Think of it as Google Drive, but specifically designed for code — it keeps every version of every file ever saved, tracks who changed what, and lets teams collaborate without overwriting each other's work.

GitHub also has a free feature called GitHub Pages that turns code files into a live website anyone can visit. That's how our Sensor Network Tracker is hosted.

Think of it like: a shared drive that also doubles as a free website host, with unlimited version history.

What It Would Do for AMQA

  • Host the Sensor Network Tracker as a website at no cost
  • Store all app code with complete history of every change
  • Host additional tools and apps that AMQA staff create in the future — any HTML/JavaScript app can be pushed to GitHub and instantly becomes a website
  • Owned by the team, not any individual — the amqa-tools organization account would persist regardless of staff changes

Supabase — Cloud Database

What It Is

Supabase is a cloud database service. It would be where all the actual data for the Sensor Network Tracker lives — every sensor record, every contact, every note, every alert.

When you use the app and make changes (add a contact, write a note, update a sensor), those changes would be saved to Supabase. When anyone opens the app later, the data is pulled from Supabase and displayed.

Think of it like: a secure, always-online spreadsheet in the cloud that only authorized apps and users can access.

Can Other AMQA Tools Use Supabase?

Yes. A single Supabase project can host multiple apps. Each app stores its data in separate tables, so they don't interfere with each other. If the team builds new tools (analysis dashboards, data collection forms, internal trackers), they can all use the same Supabase project at no additional cost.

Alternatively, you can create entirely separate Supabase projects for different tools — each free project gets its own database.

Claude Code — AI Development Tool

What It Is

Claude is an AI assistant made by Anthropic. The Claude Team Plan gives team members access to Claude's AI capabilities — including Claude Code, the tool used to build the Sensor Network Tracker prototype. It would enable any team member to build new tools, fix issues, create analysis scripts, generate reports, and more.

You interact with it by describing what you want in plain English, and it writes the code, creates the files, and tests that everything works. No prior coding experience is required.

Think of it like: having a software developer on call 24/7 who builds whatever you ask for and never takes vacation.

What It Can Do for AMQA

  • Maintain the Sensor Network Tracker — fix bugs, add features, update integrations

  • Connect to existing tools — integrates with Google Drive, Gmail, Calendar, GitHub, Microsoft 365, and Slack

How R, Shiny App, and This Proposal Fit Together

What Is R?

R is a programming language designed for data analysis and statistics. ADEC staff already use R to analyze air quality data — cleaning datasets, running calculations, creating charts, and generating reports.

Think of it like: a very powerful calculator that can process thousands of rows of data and make charts automatically.

What Is Shiny App (Connect Cloud)?

Shiny is a tool built on top of R that turns R scripts into interactive web apps. Instead of running an R script on your computer and looking at the output, Shiny lets you put that analysis on a website where people can click buttons, choose options, and see results update in real time.

Connect Cloud (formerly Shiny hosting) is the paid service that hosts those Shiny apps online so others can access them.

Think of it like: R does the math, Shiny puts a website wrapper around it so other people can use it without knowing R.

Why Shiny Can't Host the Sensor Network Tracker

This is an important distinction: Shiny apps and the Sensor Network Tracker are built in completely different programming languages.

Shiny AppsSensor Network Tracker
Written inR (a statistics language)HTML, CSS, JavaScript (web languages)
Designed forData analysis dashboardsFull-featured web applications
Hosted onConnect Cloud (paid R hosting)GitHub Pages (free web hosting)
Can they swap?No — like trying to open a Word document in a music player. They're different formats entirely.

Rebuilding the Sensor Network Tracker in R/Shiny would mean starting from scratch — months of work, and Shiny isn't designed for the kind of app this is (contact management, note-taking, sensor tracking with history). Shiny is great for data dashboards, but not for full applications with user accounts, databases, and complex workflows.

How R and This Proposal Work Together

The good news: R and the proposed toolset complement each other perfectly. They don't compete — they serve different purposes:

TaskBest ToolWhy
Analyzing air quality dataRR is purpose-built for statistics and data analysis
Creating data analysis dashboardsR + ShinyShiny makes interactive charts and reports
Tracking sensors, contacts, and communitiesSensor Tracker (GitHub + Supabase)Full app with database, history, alerts
Building custom workflow toolsClaude Code + GitHubBuild any web tool and host it for free
Quick data processing scriptsR or Claude CodeEither works — use whichever the person is comfortable with

They can even connect: An R script could pull data from the Supabase database for analysis, and a web tool built with Claude Code could display results from R analyses. The tools work alongside each other, not instead of each other.

Bottom line: Keep using R and Shiny for what they're great at (data analysis and dashboards). Use the proposed GitHub + Supabase + Claude Code toolset for everything else (app hosting, databases, workflow tools, and building new tools on demand). There's no need to choose one or the other.

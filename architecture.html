How the Sensor Network Tracker Works

A plain-language guide for anyone — no technical background needed

What Is This App?

The Sensor Network Tracker is a website that helps ADEC manage air quality sensors across Alaska. It tracks which sensors are where, who the contacts are at each community, and whether the sensors are working properly.

You access it by visiting this link in any web browser:

https://amqa-tools.github.io/platform/

The Three Pieces

The app is made up of three separate services that work together:

🌐

The App Itself

A collection of files that tell your web browser what to show on screen and how to behave when you click buttons. When you visit the app's web address, your browser downloads these files and runs them — like how a PDF opens in your browser. No special software to install.

These files are hosted for free on GitHub Pages, which turns them into a website anyone with the link can visit.

Think of it like: a set of instructions your browser follows to build the app on your screen.
🗃

The Database (Supabase)

All the actual information — every sensor, every contact, every note, every alert — is stored in an online database called Supabase.

When you add a new contact or write a note, the app saves it to Supabase. When you open the app next time, it pulls everything back out to show you.

What's stored there: sensor records, contacts, communities, notes, communication logs, alerts, uploaded files, and login accounts.

Think of it like: a secure filing cabinet in the cloud that only the app can open.
💾

GitHub — Code Storage

GitHub stores the app's files with a complete history of every change ever made. It records who changed what, when, and why — so nothing is ever truly lost.

The account is owned by the amqa-tools organization, not any individual person. If someone leaves, the organization still owns everything.

Think of it like: a shared drive that keeps every version of every document forever.

The Key Thing to Understand

GitHub stores the app. Supabase stores the data. Your browser brings them together.

QuantAQ Sensor Health Checks

The app can check whether your air quality sensors are working properly by talking to QuantAQ (the sensor manufacturer's system).

When you click "Run QuantAQ Check," the app asks QuantAQ: "How are all our sensors doing?" QuantAQ sends back status info, and the app looks for problems (sensor offline, readings look wrong, etc.).

Smart filtering built in: Not every flag means something is actually wrong. For example, after a power outage, gas sensors need hours to warm up and will look broken even though they're fine. The app now waits a grace period before creating an alert:

  • Gas sensor flags: 6-hour waiting period
  • Lost connection: 2-hour waiting period
  • PM sensor or SD card issues: Alert created immediately (these are real hardware problems)

If the issue goes away on its own during the waiting period, no alert is ever created. This prevents a flood of false alarms from routine power cycles and sensor warmups.

What Happens If Someone Leaves ADEC?

The app keeps running. Nobody needs to do anything for it to continue working day-to-day. It's not running on anyone's personal computer — it's hosted on the internet.

Specifically:

  • The website stays live at the same URL — no action needed
  • All data stays in Supabase — no action needed
  • The code stays on GitHub, owned by the organization — no action needed
  • QuantAQ checks keep working as long as the QuantAQ subscription is active

What someone would eventually need to do:

  • If you want new features or need to fix something, you'd need someone who can edit the code (a web developer, or someone using an AI coding tool)
  • Keep the Supabase account active (free right now; $25/month if you want automatic backups)
  • Keep the QuantAQ subscription active for sensor health checks

Backing Up Your Data

The code is already backed up — GitHub keeps every version of every file ever saved.

The data (sensors, contacts, notes) has a few backup options:

MethodHowCost
Export from the appUse the "Export" button on the contacts or sensors page to download a spreadsheetFree
Supabase dashboard exportLog into the Supabase website and export the full databaseFree
Automatic daily backupsSupabase Pro plan backs up everything daily, with the ability to restore to any point in time$25/month

Recommendation: If losing this data would be a serious problem, the $25/month plan for automatic daily backups is worth it. Otherwise, periodic manual exports give you a safety net at no cost.
